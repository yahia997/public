# What is a VCS (Version Control System) ?
Let's assume you made some edits on a project for a client. The client said: <i>"I love those changes"</i>. But, unfortunately, after 2 days, the client changed his opinion and wanted the old version. How would you bring back the old version? You have already overwritten the old code!

Then the only possible solution in this case is to fix it manually again. (No `Ctrl+Z` as you closed the code editor, and it is not practical to keep a local copy of your code every time you modify the code.

Here, the **VCS (Version control system)** comes into play. It saves your **history of all of your modifications**, so you can return to any point you want and track changes over time.

Not only did it provide the ability for other **developers to collaborate** effectively on the project, using features like(we will cover later, do not overwhelm yourself with those concepts now):
- Branching.
- Merging.
