# What are Git & GitHub?

Everyone talks about Git and GitHub — but they're not the same thing. In this phase, we'll cover:

1. What Git and GitHub actually are
2. The key differences between them

## Git

As we covered in the previous phase (*"What is a VCS?"*), Git is a Version Control System — not the
only one out there, but by far the most widely used. Different VCS tools vary in the details, but they
all share the same core idea: **tracking changes over time.**

Because Git is a VCS, it gives you the same core benefits:
- Keeps a **history of every modification** you make
- Enables **efficient collaboration** between multiple developers

Git itself runs entirely on your machine — no internet connection required. It's a tool, not a website.

## GitHub

So where does GitHub fit in?

Imagine you and a friend are working on the same project at the same time. You make some changes,
they make different changes — now there are two separate versions of the project, each missing
what the other person did. Someone needs a way to bring those versions back together.

That's exactly what **GitHub** helps with. Think of it as an **online platform for storing your Git
project** — a shared home where everyone's changes can be pushed, pulled, and merged together
into one consistent version. It also lets you control who has access, so you can invite other
developers to view or contribute to your project.

## Git vs. GitHub — the short version

| Git | GitHub |
|---|---|
| A tool installed on your computer | A website/platform built around Git |
| Tracks history locally | Hosts your project online for others to access |
| Works with or without internet | Requires internet access |
| Created by Linus Torvalds (2005) | A company/service (owned by Microsoft) |

> **💡 In short:** Git is *how* you track changes. GitHub is *where* you share them.
