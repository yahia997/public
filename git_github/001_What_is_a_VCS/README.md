# What is a VCS (Version Control System)?

Imagine you've been working on a project for a client. You make some changes, and the client says:
*"I love those changes!"* Two days later, they change their mind and want the old version back.

How do you bring it back? You've already overwritten the old code — there's no `Ctrl+Z` waiting for
you, especially after you've closed your editor. And realistically, keeping a separate saved copy of
your code every time you make a change isn't practical either.

This is exactly the problem a **Version Control System (VCS)** solves. A VCS keeps a complete
**history of every change** made to your project, so you can go back to any previous point in time
whenever you need to — no guesswork, no manual rewrites, no lost work.

But a VCS isn't just a safety net for undoing mistakes. It's also what makes it possible for **multiple
developers to collaborate** on the same project smoothly, using concepts like:

- **Branching** — working on separate lines of development without interfering with each other
- **Merging** — combining that work back together once it's ready

*(Don't worry about fully understanding branching and merging yet — we'll dive into both in detail
in a later phase.)*
